A good modern web browser

Of all the browser options, Chrome and Firefox are usually at the forefront of developing web technology, such as having support for things like audio signal processing, 3D graphics, VR.

Here's Firefox; made by the not-for-profit Mozilla Foundation.

Here's Chrome, and here's the near-identical open-source alternative, Chromium, both made by Google.

Like having a development environment in the cloud, these websites allow you to quickly code up HTML/CSS/Javascript browser code and see the results immediately.

Visual Studio Code

A not-so-heavy code editor with many integrated features (including a command line terminal). (Not to be confused with Visual Studio, which is a behemoth enviornment primarily oriented to C++ and C# development on Windows.)

Why VSCode?

  • It behaves nearly identically on Windows, OSX, Linux, etc.
  • At present it is the most loved development environment.
  • Super configurable: Lots of plugins, you can set up build tasks, etc.

Download here. Among the install options, if it asks to add features to the Finder/Explorer window, I recommend enabling them. Similarly if it asks to add code to the PATH, I recommend this too.

Node.js and NPM

Node.js is an event-driven environment oriented toward command-line apps and servers, written in JavaScript. With Node.js we can easily write servers to generate static and dynamic web-pages, which also talk to other services in the operating system, like reading & writing files, databases, sensor networks, etc. NPM is Node's package manager, a way to easily define and install additional library dependencies to extend the capabilities of Node, and to automate build processes of your own projects.

Node.js installers can be downloaded here -- I recommend the LTS (Long Term Service) version. Installing Node with default options will also install NPM. Be sure to select the option to add Node and NPM to your PATH (so that you can use them from the command line).

Git and Github

Git is a "version control" system (VCS); essentially a way to keep a history of edits made to a code-base (AKA repositories). It's not the only VCS -- there's also svn, hg, etc. But git is the most widely-used.

Download & run the Git command line tools installer from here.

Github is an online cloud for storing and sharing code repositories. It's also not the only one, there's Bitbucket, Gitlab, Sourceforge etc. However Github has historically had the largest number of open-source repositories on it, and has a particularly nice feature that it also supports static web hosting (simply by creating a branch in your repository with the name gh-pages), which will be useful for us.
